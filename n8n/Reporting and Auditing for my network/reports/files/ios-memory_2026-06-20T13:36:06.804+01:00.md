# Network Assessment Report

## Executive Summary
Memory utilization was collected from all IOS/IOS-XE/IOS-XR devices in the inventory. All four devices (`R1`, `R2`, `SW1`, `SW2`) are IOS-XE platforms. **All devices report memory utilization well below the 80% threshold**, with values ranging between approximately **13.1% and 15.7%**. No alerts are raised.

## Scope & Assumptions
- **Scope**: All devices in the pyATS testbed inventory matching OS types `ios`, `iosxe`, or `iosxr`.
- **Inventory**: 4 devices total, all `iosxe` (no `ios` or `iosxr` devices present).
- **Data source**: `show memory statistics` (Processor pool) ÔÇö `show processes memory statistics` is not supported on this IOL platform.
- **Threshold**: Issue raised only if utilization exceeds **80%**.
- **Assumption**: Processor memory pool is representative of overall device memory utilization for these IOL-based devices.

- **No device exceeds the 80% threshold** ÔÇö no issues are raised per the operating notes.

## Risks & Considerations
- The MCP command `show processes memory statistics` is unsupported on these IOL images; per-process granularity is not available without alternative commands such as `show processes memory sorted`.
- Inventory contains only IOS-XE virtual (IOL) routers/switches ÔÇö there is **no real IOS-XR or classic IOS data** to compare against.
- Lowest-watermark values (`Lowest(b)`) are very close to the current `Free(b)`, indicating stable memory behavior with no recent pressure events.

## Recommendations
- **No immediate action required** ÔÇö all devices are healthy with respect to the 80% memory threshold.
- Continue periodic monitoring (e.g., baseline trend collection) to detect gradual leaks.
- If finer process-level visibility is needed, use `show processes memory sorted` on IOS-XE devices.
- Add IOS-XR and physical IOS devices to the testbed if broader coverage of the multi-OS estate is required.

## Conclusion
All four IOS-XE devices in the inventory report **healthy memory utilization between 13% and 16%**, well under the 80% alert threshold. No issues are flagged. The environment is currently stable from a memory perspective.
