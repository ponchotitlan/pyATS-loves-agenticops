# Network Assessment Report

## Executive Summary
Memory utilization was collected from all four IOS-XE devices in the inventory (`R1`, `R2`, `SW1`, `SW2`). All devices are operating well within healthy memory thresholds, with utilization ranging between **~13%** and **~16%**. **No device exceeds the 80% threshold**, and therefore no issues are raised.

## Scope & Assumptions
- Scope covers all devices listed in the pyATS testbed inventory.
- Inventory contains only **IOS-XE** devices (`R1`, `R2`, `SW1`, `SW2`). No IOS or IOS-XR devices were present in the inventory.
- Memory utilization is calculated from `show memory statistics` (Processor pool) as: `Used / Total * 100`.
- Threshold for raising an issue: **> 80% memory utilization**.

**Key observations:**
- Both routers (`R1`, `R2`) report nearly identical memory footprints, consistent with similar roles/configurations.
- Both switches (`SW1`, `SW2`) also show matching utilization, indicating a balanced workload.
- All `Lowest(b)` watermarks remain very close to current `Free(b)`, indicating **no historical memory pressure events**.
- `Largest(b)` contiguous free block is large on every device, indicating **no memory fragmentation concerns**.

## Risks & Considerations
- No memory-related risks identified at this time — all devices are below 20% usage.
- The inventory does **not** include any **IOS** or **IOS-XR** devices; if such devices exist outside this testbed, they would need to be added to the inventory for full coverage.
- IOL (IOS-On-Linux) platforms typically reflect host-allocated memory, which may differ from physical hardware behavior in production.

## Recommendations
- **No immediate action required.** Memory is healthy across all monitored devices.
- Continue periodic monitoring (e.g., daily/weekly) using `show memory statistics` or SNMP/telemetry.
- Consider adding **IOS** and **IOS-XR** devices to the pyATS testbed if they exist in the broader network, to ensure complete inventory coverage.
- Establish proactive alerting at **70%** (warning) and **80%** (critical) utilization thresholds.

## Conclusion
All four IOS-XE devices in the inventory show **healthy memory utilization between 13% and 16%**, well below the 80% threshold. **No issues are raised.** The environment is currently stable from a memory resource perspective.
